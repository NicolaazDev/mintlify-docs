## Autenticação

Todas as rotas protegidas da API são autenticadas usando tokens JWT (JSON Web Token). Os tokens devem ser incluídos no cabeçalho de requisição usando o formato Bearer.

```bash
Authorization: Bearer seu_token_jwt
```

## Recursos de Autenticação

O sistema oferece diversos recursos avançados de segurança:

### Registro e Verificação

- Registro de usuários com validação de email via OTP
- Verificação em duas etapas para ativação de contas

### Login Seguro

- Autenticação via email/senha
- Autenticação em dois fatores (2FA) com aplicativo autenticador
- Login social integrado com:
  - Google
  - Discord
  - Facebook
  - Steam
  - Twitch

### Recuperação de Conta

- Sistema de recuperação de senha por email
- Verificação de código OTP para reset de senha
- Políticas de senha segura

### Controle de Acesso

- Sistema de permissões baseado em roles
- Proteção de rotas por nível de acesso
- Guardas personalizados para validação de permissões

## Como Usar

Todas as chamadas à API devem ser feitas com o Content-Type `application/json`. Para rotas autenticadas, inclua o token JWT no cabeçalho de autorização.

Os exemplos abaixo mostram como utilizar os principais endpoints:

```typescript
// Exemplo de registro de usuário
const response = await fetch('/api/auth/register', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'usuario@exemplo.com',
    password: 'Senha@123',
    name: 'Nome do Usuário'
  })
});

// Exemplo de login
const loginResponse = await fetch('/api/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'usuario@exemplo.com',
    password: 'Senha@123'
  })
});
```
